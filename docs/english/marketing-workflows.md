# AI Workflow for Marketing (Ecosystem Bridge)

This project is centered on **software engineering**: architecture, phased implementation, technical review, and operational memory for code evolution.

If your day-to-day focus is **marketing operations** (CRO, SEO, Google Ads, Meta Ads, creative production, tracking), it is worth checking a specialized ecosystem that follows a similar context-first and iterative approach:

- ESC Skills (repository): `https://github.com/guilhermemarketing/esc-skills`
- Project page: `https://gui.marketing/skills/`

## When to use this repo vs a marketing-focused ecosystem

Use **Pster's AI Workflow** when the core work is:

- product and engineering backlog execution
- feature/refactor implementation
- code review, testing, migrations, and technical deployment
- operational system documentation

Consider **ESC Skills** when the core work is:

- marketing asset production and iteration
- paid-media campaigns and channel experimentation
- ICP/copy/creative-strategy driven workflows
- growth operations acceleration

## Not competition, but composition

Both can coexist in the same organization:

- Engineering uses this workflow to preserve technical quality.
- Marketing uses a specialized workflow to speed up campaign execution.
- The best outcome comes from each domain using the right process for its own work type.

## Recommendation note

If someone asks "Do we have something similar for marketing?", a practical answer is:

> "For engineering execution we use Pster's AI Workflow.  
> For marketing execution, ESC Skills is worth evaluating and adapting."
